title: Выбор технологий для реализации программного обеспечения
---

При выборе средств разработки инструмента нефункционального тестирования безопасности REST API ProofAPI учитывались функциональные требования к системе, а также необходимость обеспечения кроссплатформенности, высокой производительности при работе с сетевыми запросами, удобства сопровождения и расширяемости. Разрабатываемое программное обеспечение должно выполнять следующие основные задачи: импорт API-спецификаций в формате OpenAPI, нагрузочное тестирование с имитацией множества виртуальных пользователей, тестирование безопасности методами фаззинга с генерацией вредоносных нагрузок, проверку заголовков безопасности и утечек конфиденциальных данных, локальное хранение результатов с возможностью версионирования и сравнения, а также формирование детализированных HTML-отчётов с графиками и cURL-командами для воспроизведения найденных уязвимостей. Исходя из этих требований, были сформулированы следующие ключевые критерии отбора технологий: поддержка современных парадигм программирования и асинхронной обработки, наличие развитых библиотек для работы с HTTP и OpenAPI, возможность создания настольного интерфейса, работающего на Windows, Linux и macOS, а также наличие средств для удобного объектно-реляционного отображения и ведения локальной базы данных без необходимости развёртывания серверной СУБД. Дополнительно принималась во внимание доступность инструментов и лицензионная чистота всего используемого программного обеспечения.
