#### **Действующие лица**

В системе выделены следующие действующие лица (actors):

-  **Тестировщик / Специалист по безопасности** – основной пользователь, который импортирует спецификацию, настраивает и запускает тесты, анализирует отчёты.

-  **Разработчик (Dev)** – может запускать тесты из CI/CD (в автоматическом режиме) для проверки API перед релизом.

-  **Администратор системы** – управляет учётными записями (если будет внедрена многопользовательская версия) и архивацией старых результатов.

Для минимально жизнеспособного продукта (MVP) рассматривается только роль **Тестировщика**. Остальные роли могут быть добавлены в будущем.



#### **Диаграмма вариантов использования**

Диаграмма вариантов использования для роли «Тестировщик» представлена на рисунке 1 (будет добавлен). Основные варианты:

-  Загрузить OpenAPI‑спецификацию

-  Сконфигурировать аутентификацию

-  Настроить профиль нагрузки

-  Выбрать типы проверок безопасности

-  Запустить нагрузочный тест

-  Запустить сканирование безопасности

-  Просмотреть статус выполнения (progress)

-  Сохранить проект

-  Просмотреть историю запусков

-  Сравнить два прогона (регрессия)

-  Сгенерировать отчёт (HTML/JSON)

-  Воспроизвести уязвимость (получить cURL)



#### **Описание вариантов использования (пример)**

**Таблица 4 – Вариант использования «Запустить сканирование безопасности»**

| **Атрибут**                  | **Значение**                                                                                                                                                                                                                                                                                                                                                                    |
|------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **ID**                       | UC‑03                                                                                                                                                                                                                                                                                                                                                                           |
| **Название**                 | Запустить сканирование безопасности                                                                                                                                                                                                                                                                                                                                             |
| **Краткое описание**         | Пользователь инициирует процесс проверки API на наличие уязвимостей (SQLi, XSS, BOLA, утечки данных).                                                                                                                                                                                                                                                                           |
| **Главное действующее лицо** | Тестировщик                                                                                                                                                                                                                                                                                                                                                                     |
| **Предусловия**              | Проект загружен, OpenAPI‑спецификация импортирована, аутентификация настроена.                                                                                                                                                                                                                                                                                                  |
| **Основной поток**           | 1\. Пользователь выбирает пункт меню «Запустить проверку безопасности». 2. Система отображает диалог выбора типов атак и параметров фаззинга. 3. Пользователь подтверждает запуск. 4. Система последовательно (или параллельно) отправляет запросы с payload'ами, анализирует ответы. 5. По завершении система выводит сводку: количество найденных уязвимостей по критичности. |
| **Постусловия**              | Результаты сохранены в локальной БД, доступен предварительный просмотр уязвимостей.                                                                                                                                                                                                                                                                                             |
| **Альтернативный поток**     | При отсутствии аутентификации или спецификации система выводит предупреждение и не запускает сканирование.                                                                                                                                                                                                                                                                      |

(Аналогично описываются варианты «Настроить нагрузочный тест», «Сгенерировать отчёт», «Сравнить два запуска».)
